Model Performance Overview

Comprehensive evaluation metrics for the Hybrid Fraud Detection Model (XGBoost + Isolation Forest)

Accuracy

Overall Correctness

--
Percentage of correct predictions (both fraud and genuine) out of all predictions made.
Formula: (TP + TN) / (TP + TN + FP + FN)

Precision

Positive Predictive Value

--
Of all claims predicted as fraud, what percentage were actually fraudulent?
Formula: TP / (TP + FP)

Recall

Sensitivity / True Positive Rate

--
Of all actual fraudulent claims, what percentage did the model successfully detect?
Formula: TP / (TP + FN)

F1-Score

Harmonic Mean of Precision & Recall

--
Balanced measure that combines precision and recall into a single metric.
Formula: 2 × (Precision × Recall) / (Precision + Recall)

ROC-AUC

Area Under ROC Curve

--
Measures the model's ability to distinguish between fraudulent and genuine claims.
Interpretation: 1.0 = Perfect, 0.5 = Random, <0.5 = Worse than random

📊 Dataset Information

Total Samples: 5,000 claims
Training Set: 4,000 claims (80%)
Test Set: 1,000 claims (20%)
Class Distribution: 50% Fraud / 50% Genuine

🤖 Model Architecture

Primary Model: XGBoost Classifier (70% weight)
Secondary Model: Isolation Forest (30% weight)
Hybrid Approach: Weighted combination
Decision Threshold: 0.5 (50%)

🎯 Feature Engineering

Total Features: 13 features
Preprocessing: StandardScaler normalization
Explainability: SHAP values
Cross-validation: Stratified split

Understanding the Metrics

TP
True Positive: Correctly identified fraud cases
TN
True Negative: Correctly identified genuine cases
FP
False Positive: Genuine claims incorrectly flagged as fraud (Type I Error)
FN
False Negative: Fraudulent claims missed by the model (Type II Error)
